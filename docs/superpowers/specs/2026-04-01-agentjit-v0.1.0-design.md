# AgentJIT v0.1.0 вЂ” Design Specification

**Status:** Approved
**Date:** 2026-04-01
**Tech Stack:** Go (single binary), Claude Code (compiler), JSONL (storage)

---

## 1. Overview

AgentJIT is a background JIT compiler for autonomous coding agents. It silently ingests execution telemetry (tool calls, shell commands) from Claude Code sessions, identifies recurring multi-step patterns, and compiles them into zero-token parameterized skills with measurable ROI metrics.

### Value Proposition

- **Token optimization:** Replaces 10,000+ token stochastic reasoning loops with ~200 token deterministic skill invocations.
- **Latency reduction:** Drops execution time for routine tasks from >30s to <1s.
- **Zero friction:** Operates entirely in the background via Claude Code hooks. No manual configuration.

---

## 2. Architecture

Three layers, loosely coupled:

1. **Ingestion Layer (Go daemon)** вЂ” Intercepts Claude Code hook events, normalizes them, appends to partitioned JSONL logs.
2. **Trigger Layer (Go daemon)** вЂ” Monitors event counts or timers, fires the dream compilation sequence.
3. **Compiler Layer (Claude CLI)** вЂ” Claude Code itself reads the logs, identifies patterns, generates/updates skills.

The Go binary is a "dumb pipe" вЂ” it handles I/O, lifecycle, and configuration. All intelligence lives in the compiler prompt that Claude executes.

## 5. Event Ingestion

### 5.1 Hook Configuration

`agentjit init` installs four hooks into Claude Code's `~/.claude/settings.json`:

**PostToolUse** (async):
```json
{
  "hooks": [
    {
      "type": "command",
      "command": "agentjit ingest",
      "async": true
    }
  ]
}
```

**PostToolUseFailure** (async):
```json
{
  "hooks": [
    {
      "type": "command",
      "command": "agentjit ingest",
      "async": true
    }
  ]
}
```

**SessionStart** (synchronous вЂ” ensures daemon is running, injects context via stdout):
```json
{
  "hooks": [
    {
      "type": "command",
      "command": "agentjit daemon start --if-not-running && agentjit ingest"
    }
  ]
}
```

The `SessionStart` hook outputs JSON with `additionalContext` on stdout: `"[AgentJIT] Ingestion active. Daemon PID <pid>. Dream mode: <mode>."` This appears in Claude's context for the session.

**SessionEnd** (async):
```json
{
  "hooks": [
    {
      "type": "command",
      "command": "agentjit ingest",
      "async": true
    }
  ]
}
```

Hook installation is non-destructive вЂ” it merges into existing hook arrays without clobbering other tools' hooks.

### 5.2 The `agentjit ingest` Command

1. Reads JSON from stdin (Claude Code hook payload).
2. Enriches with normalized fields (`harness: "claude-code"`, normalized `event_type`).
3. Forwards to the running daemon via Unix domain socket at `~/.agentjit/daemon.sock`.
4. Fallback: if daemon is unreachable, appends directly to the JSONL log file.

### 5.3 Normalized Event Schema

```json
{
  "timestamp": "2026-04-01T05:28:45Z",
  "session_id": "cld_abc123",
  "harness": "claude-code",
  "event_type": "post_tool_use",
  "tool_name": "Bash",
  "tool_input": {
    "command": "kubectl logs -n eventstream pod/ingest-xyz"
  },
  "tool_response_summary": "<truncated to max_response_bytes>",
  "exit_code": 0,
  "working_directory": "/Users/dev/workspace",
  "metrics": {
    "execution_duration_ms": 4200
  }
}
```

- `tool_response_summary` is truncated to `max_response_bytes` (default 512) to avoid log bloat.
- Token metrics are not available in hook payloads. The dream compiler estimates token costs from input/response sizes.
- `SessionStart` events include `source` (startup/resume/clear). `SessionEnd` events include `session_duration_ms`, `num_turns`.

## 6. Daemon Architecture

### 6.1 Components

The daemon is a single Go process with three goroutines:

1. **Unix socket server** вЂ” Listens on `~/.agentjit/daemon.sock`. Receives events from `agentjit ingest`, validates, and appends to the correct JSONL file. Maintains an in-memory event counter.

2. **Dream trigger monitor** вЂ” Evaluates whether to fire the dream sequence based on configured mode:

| Mode | Behavior | Config Keys |
|---|---|---|
| `manual` (default) | No automatic trigger | `trigger_mode: "manual"` |
| `interval` | Fires every N minutes | `trigger_mode: "interval"`, `trigger_interval_minutes: 30` |
| `event_count` | Fires after N new events since last dream | `trigger_mode: "event_count"`, `trigger_event_threshold: 100` |

Only one dream can run concurrently. Additional triggers are dropped with a log message.

3. **Skill file watcher** вЂ” Uses `fsnotify` to watch `~/.agentjit/skills/` and registered project-local `.claude/skills/` directories. On new skill detection, emits:
```
[AgentJIT] Compiled skill: 'get-aks-pod-logs'. Estimated savings: 18,300 tokens/invocation.
```

### 6.2 Lifecycle

- **Auto-start:** `SessionStart` hook runs `agentjit daemon start --if-not-running`. Zero developer action needed.
- **Idle timeout:** After 30 minutes (configurable) with no events, the daemon shuts down. Next `SessionStart` restarts it.
- **PID management:** PID written to `~/.agentjit/daemon.pid`. Stale PID files (process no longer alive) are cleaned up automatically.
- **Graceful shutdown:** `agentjit daemon stop` sends `POST /shutdown` to the socket. Daemon cleans up PID file, socket, and exits.
- **No port conflicts:** Unix domain socket eliminates TCP port collision issues.

## 7. Dream Compiler

### 7.1 Execution Flow

`agentjit dream` orchestrates the reflection phase:

1. **Gather logs** вЂ” Scan `~/.agentjit/logs/` for session files newer than `~/.agentjit/last_dream_marker`.

2. **Gather existing skills** вЂ” Read all skill files in `~/.agentjit/skills/` and registered project-local directories. Extract metadata: name, description, parameters, ROI, creation date, pattern hash.

3. **Build context** вЂ” Concatenate unprocessed events (grouped by session) and existing skill inventory into a temporary context file. Enforce `max_context_lines` (default 50,000) вЂ” prioritize recent sessions if exceeded.

4. **Invoke Claude** вЂ” Shell out to:
   ```
   claude --print -p "$(cat prompts/compiler.md)" \
     --allowedTools Read,Write,Bash,Glob,Grep \
     < /tmp/agentjit-dream-context.jsonl
   ```

5. **Update marker** вЂ” Write current timestamp to `last_dream_marker`.

6. **Audit log** вЂ” Append a summary of proposed and written changes to `~/.agentjit/dream-log.jsonl`.

7. **Notification** вЂ” The daemon's file watcher picks up new skill files and emits notifications.

### 7.2 Compiler Prompt Responsibilities

The compiler prompt (`prompts/compiler.md`) instructs Claude to:

- **Identify patterns:** Find sequences of 2+ tool calls appearing across 3+ sessions with the same logical structure but varying parameters.
- **Parameterize:** Identify hardcoded vs. dynamic values. Lift dynamic values into script arguments.
- **Infer scope:** Apply layered heuristic:
  1. Same sequence in 2+ distinct `working_directory` roots в†’ global (`~/.agentjit/skills/`)
  2. Only in one project в†’ local (`.claude/skills/` in that project)
  3. Fallback: primarily uses global CLIs (kubectl, az, gh, docker, aws, gcloud, terraform, helm) в†’ global
  4. Write inferred scope and reasoning into skill metadata for user override
- **Calculate ROI:** For each pattern:
  - `stochastic_cost = avg_input_tokens + avg_output_tokens` (estimated from input/response sizes)
  - `deterministic_cost = ~200 tokens` (skill invocation overhead)
  - `savings_per_invocation = stochastic_cost - deterministic_cost`
  - `total_projected_savings = savings_per_invocation * observed_frequency`
- **Apply thresholds:** Skip patterns below `min_pattern_frequency` (default 3) AND `min_token_savings` (default 500).
- **Manage existing skills:**

| Action | Trigger |
|---|---|
| Create new skill | New pattern exceeds thresholds |
| Optimize existing skill | Pattern data suggests more general parameters, fewer steps, or missing edge cases |
| Merge skills | Two skills frequently called in sequence |
| Deprecate skill | Not invoked in N sessions (default 20) or pattern has drifted |
| Version skill | When updating, rename old to `<name>.v1.md` as backup |

- **Output action plan** before writing files:
  ```
  ## Proposed Changes
  - NEW: create-k8s-deployment (savings: 12,400 tokens/invocation, frequency: 7)
  - UPDATE: get-aks-pod-logs v1->v2 (added --since flag, +200 tokens savings)
  - DEPRECATE: check-pod-status (last used 25 sessions ago)
  ```

### 8.3 Companion Script

```bash
#!/usr/bin/env bash
set -euo pipefail

# Generated by AgentJIT вЂ” do not edit manually
# Source pattern: get-credentials -> kubectl logs
# Self-healing: exits 2 on auth failure to hand back to LLM

NAMESPACE="${1:?Usage: get-aks-pod-logs.sh NAMESPACE POD_NAME [SINCE]}"
POD_NAME="${2:?Usage: get-aks-pod-logs.sh NAMESPACE POD_NAME [SINCE]}"
SINCE="${3:-}"

az aks get-credentials --resource-group mygroup --name mycluster 2>/dev/null

SINCE_FLAG=""
if [ -n "$SINCE" ]; then
  SINCE_FLAG="--since=$SINCE"
fi

if ! kubectl logs -n "$NAMESPACE" "pod/$POD_NAME" $SINCE_FLAG 2>/tmp/agentjit-err; then
  ERR=$(cat /tmp/agentjit-err)
  if echo "$ERR" | grep -qi "unauthorized\|forbidden\|token"; then
    echo "Auth failure detected вЂ” handing back to agent" >&2
    exit 2
  fi
  echo "$ERR" >&2
  exit 1
fi
```

Key conventions:
- `set -euo pipefail` for safety
- Exit code 2 triggers self-healing (Claude Code receives stderr and can resolve)
- `generated_by: agentjit` distinguishes from hand-written skills
- `source_pattern_hash` links to the originating log pattern

## 9. Bootstrap

### 9.1 Purpose

Import historical Claude Code session transcripts into AgentJIT's JSONL log format, giving the dream compiler a head start before any hooks have fired.

### 9.2 Flow

1. **Locate transcripts** вЂ” Claude Code stores session data in `~/.claude/projects/`. Each session has a transcript JSON containing tool calls, inputs, and outputs.
2. **Invoke Claude** вЂ” Shell out to `claude` with an extraction prompt. Claude reads the transcripts and outputs normalized JSONL events matching the ingestion schema.
3. **Write to logs** вЂ” Output goes to `~/.agentjit/logs/` with date/session partitioning derived from transcript timestamps. Events are tagged with `"source": "bootstrap"`.
4. **Track processed files** вЂ” `~/.agentjit/bootstrap_processed.json` records which transcripts have been processed. Running `agentjit bootstrap` again only picks up new sessions.
5. **Prompt for dream** вЂ” After completion: `"Bootstrapped 47 sessions (2,340 events). Run dream compilation now? [Y/n]"`

### 9.3 CLI Flags

| Flag | Behavior |
|---|---|
| `--since DATE` | Only transcripts after the given date |
| `--project PATH` | Only transcripts for a specific project directory |
| `--dry-run` | Show what would be processed without writing |

## 11. Init Flow

`agentjit init` performs:

1. Create directories: `~/.agentjit/`, `~/.agentjit/logs/`, `~/.agentjit/skills/`
2. Write default `~/.agentjit/config.json`
3. Install hooks into `~/.claude/settings.json` (non-destructive merge)
4. Verify `agentjit` is on PATH (warn if not)
5. Print summary and suggest `agentjit bootstrap`

Flags:
- `--local` вЂ” Install hooks into `.claude/settings.json` in the current project, create `.claude/skills/` for local skill output.
- `--uninstall` вЂ” Remove AgentJIT hooks from settings, stop daemon, optionally remove `~/.agentjit/` (with confirmation).

## 12. Out of Scope (v0.1.0)

- Active authentication management (pre-flight auth checks)
- Cross-machine syncing of `~/.agentjit/`
- Custom LLM endpoints (uses host agent's API connection)
- GUI or web dashboard
- Support for agents other than Claude Code (Copilot CLI adapter deferred)
